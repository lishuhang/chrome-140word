使用说明

概述

本插件用于在浏览器中打开网页时，使用 AI 模型对网页内容生成约 140 字的一句话摘要。插件在浏览器启动后常驻后台运行，只有当用户点击工具栏图标打开右侧边栏时，才会开始对当前及后续打开的网页进行自动摘要。

工作原理

1. 插件作为 Chrome Extension (Manifest V3) 安装后，background.js 作为 Service Worker 常驻后台。

2. 当用户点击工具栏图标打开侧边栏时，插件通过 chrome.sidePanel API 展示 sidebar.html。

3. 侧边栏打开后，通过 chrome.runtime.connect 建立持久连接（port），通知 background 开始工作。

4. Background 监听标签页变化事件，仅在 sidebar port 存在时才将新页面加入摘要队列。

5. 页面加入队列前，background 会先提取页面内容并存储。v1.7 起，提取方式为：获取浏览器中已完整渲染的页面 HTML（而非从源站抓取），通过 Turndown 引擎将其转为干净的 Markdown 格式，连同页头页脚等全部内容发送给 AI 模型。AI 根据 Markdown 的结构化信息自动识别正文主体，忽略导航、广告等非正文内容。因此用户无需为不同网站配置 CSS 选择器——AI 看到的就是你看到的。

6. 队列逐个处理：使用预提取的 Markdown 文本调用 AI API，返回摘要结果。

7. 摘要结果缓存到 chrome.storage.local（30 分钟有效），同时保存到历史记录（最多 200 条）。

摘要区域交互

摘要结果直接显示在历史记录区域。点击标题区域在新窗口打开原文，点击摘要区域一键复制。鼠标悬停时会显示对应提示遮罩。切换标签页时，正在生成中的进度条目不会消失，会持续跟踪直到完成或出错。

跳过/恢复按钮

跳过/恢复按钮始终显示在状态区与历史记录之间。对于普通网页，可点击"跳过本页"或"跳过本站"将其加入跳过名单，已跳过的页面显示"恢复本页"/"恢复本站"。v1.6 起，"跳过本站"使用顶级域名匹配，例如对 www.businessinsider.com 点击"跳过本站"会生成 *.businessinsider.com/* 规则，同时匹配 businessinsider.com、www.businessinsider.com、m.businessinsider.com 等所有子域名。对于本地文件（file://），默认跳过，显示"恢复本页"按钮（点击后一次性摘要该页，不修改跳过名单）和灰色"跳过本站"按钮。对于浏览器内部页面（chrome:// 等），两个按钮均为灰色不可用。

网站专属提示词

v1.7 对此功能进行了重大简化。v1.6 版本中，网站专属配置包含"移除选择器"和"优先选择器"两个 CSS 选择器字段，用于在 DOM 层面处理特定网站的结构问题（如 Business Insider 的 article 标签劫持问题）。然而，CSS 选择器方案存在根本缺陷：每个网站的结构不同，用户需要理解 HTML/CSS 才能正确配置，且网站改版后选择器即失效。

v1.7 采用了全新的"所见即所送"策略：将浏览器中已完整渲染的页面 HTML 通过 Turndown 引擎转为 Markdown，整页发送给 AI。AI 可以从 Markdown 的结构化信息（标题层级、列表、链接等）中自动识别正文主体，忽略导航菜单、页头页脚、广告、侧边栏等非正文内容。这一策略从根本上消除了对 CSS 选择器的依赖——AI 看到的就是用户在浏览器中实际看到的内容。

网站专属提示词现在仅保留"补充提示词"字段，用于向 AI 传达特定网站的行为约束（如"摘要中不得出现某品牌名"）。插件内置了 businessinsider.com 的提示词，你可以在设置中编辑或删除它，也可以为其他网站添加自定义配置。域名匹配规则与"跳过本站"一致：输入顶级域名即自动匹配所有子域名。

本地文件处理

本地 HTML 文件（file:// 协议）默认不生成摘要。点击"恢复本页"可一次性对该页生成摘要，摘要完成后页面仍保持默认跳过状态，不会将本地路径加入跳过名单。如需对本地文件使用此功能，需在扩展设置中开启"允许访问文件网址"权限。

核心架构

background.js — Service Worker，负责：消息路由、队列管理、页面内容提取（加入队列前即提取，两步注入：先注入 turndown.js 再注入提取函数）、API 调用（含网站专属提示词组装）、缓存与历史记录管理、跳过名单匹配（使用顶级域名）、本地文件默认跳过逻辑。注意：Turndown 不在 Service Worker 中运行（因为 Service Worker 缺少 DOMParser），而是注入到页面上下文中执行。

turndown.js — 第三方库（Turndown v7.2.4），用于将 HTML 转换为 Markdown。通过 chrome.scripting.executeScript({files: ['turndown.js']}) 注入到页面上下文，然后提取函数直接传 DOM 节点（而非 HTML 字符串）给 Turndown，绕过 DOMParser 依赖。

sidebar.html + sidebar.js + sidebar.css — 侧边栏界面，包含两个 Tab：摘要（队列状态 + 跳过/恢复按钮 + 历史记录）和设置（模型配置、Prompt、网站专属提示词、跳过名单、设置导入导出、API Key 指南、使用说明、Changelog）。

AI Debug 参考

  • 内容提取方式：v1.7 起，采用两步注入策略：①先通过 chrome.scripting.executeScript({files: ['turndown.js']}) 将 Turndown 库注入页面上下文；②再注入提取函数 extractPageAsMarkdown()，该函数克隆已渲染的 DOM（document.documentElement.cloneNode(true)），移除不可见元素，创建 Turndown 实例并直接传 DOM 节点（非 HTML 字符串）给 Turndown——当 Turndown 收到 DOM 节点时，内部仅使用 cloneNode()，完全不需要 DOMParser。此方式确保 AI 看到的内容与用户在浏览器中实际看到的一致。
  • Markdown 转换：Turndown 在页面上下文中运行（而非 Service Worker，因 Service Worker 缺少 DOMParser），配置了 .remove() 规则移除导航栏、广告、评论区、社交分享等常见非正文元素。转换后的 Markdown 会清理多余空行，并截断至 20000 字符。若 Turndown 不可用，自动降级为纯文本提取。
  • 系统提示词：v1.7 的默认提示词已包含"输入内容为网页完整 Markdown，聚焦正文主体，忽略导航/广告/页头页脚等非正文内容"的指令。网站专属提示词（仅 prompt 字段）追加在系统提示词末尾。
  • 文本预提取：v1.4 起，页面内容在加入队列前即被提取并存储在队列项的 extractedText 字段中。"已加入队列"时内容已安全捕获，用户可关闭标签页。
  • 队列状态：currentProcessingItem 跟踪正在 API 调用中的项，summarizeQueue 存储等待项。sidebar 通过 port 接收 queueStatus、joinedQueue、processingStarted 等消息。
  • 进度追踪：v1.5 起，使用 progressEntries Map 按 URL 追踪进度。切换标签页时进度条目不会消失，确保从"已加入队列"到"正在摘要"到"摘要完成"使用同一个框显示。
  • 网站专属提示词：v1.7 简化为仅 domain + prompt 两个字段（移除了 removeSelectors 和 preferSelector）。存储在 chrome.storage.sync.sitePrompts。发送 API 请求时，将 prompt 追加到 systemPrompt 末尾，以 \n\n 分隔。内置提示词在插件安装时初始化，更新版本时自动迁移旧格式并补充新的内置条目，用户可编辑或删除。
  • 跳过名单：存储在 chrome.storage.sync.skipList，匹配使用通配符转正则。"跳过本站"使用 extractRegistrableDomain 提取顶级域名生成模式，urlMatchesPattern 对此类模式特殊处理，同时匹配域名本身及其所有子域名。sidebar 的跳过/恢复按钮即时切换，与设置区文本框双向同步。
  • 本地文件：file:// 协议的页面默认跳过，"恢复本页"触发 forceSummarize（绕过跳过检查），不修改跳过名单。需开启"允许访问文件网址"权限。
  • 历史交互：标题区域点击 → chrome.tabs.create 打开原文；摘要区域点击 → navigator.clipboard.writeText 复制。
